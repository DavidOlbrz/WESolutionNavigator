Übung 1: HTTP, URI und HTML

1.1 World Wide Web

  1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von Tim Burners Lee abgelehnt?
    • Man war damals der Meinung, dass das WWW ein veraltetes Konzept ist und die Wissenschaft eigentlich schon viel weiter fortgeschritten war. Ein Hauptgrund dafür war, dass "Broken Links" möglich waren, sprich Links, die ins Leere führen.
  2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?
    • Die evolvierbarkeit des WWW, sprich das sich das WWW und die einzelnen Komponenten / Technologien (z.B. HTML, HTTP, ...) unabhängig voneinander weiterentwickeln können.
  3. Was wäre der Preis für die garantierte Verhinderung von "broken links"?
    • Durch die referentielle Integrität ist es aufwendiger, Links zu pflegen. Wenn z.B. eine Seite gelöscht wird, müssen auch alle Links entfernt werden, die auf diese Seite geführt haben, wenn eine Seite an eine andere Adresse verschoben wird, müssen alle draufzeigenden Links angepasst werden etc. Man bräuchte quasi eine zentralle Kontrolle.

1.2 HTTP

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
    • Die Anfrage wurde erfolgreich bearbeitet
  2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
    • Die angeforderte Ressource wurde permanent an eine neue URL verschoben
  3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
    • Die Anfrage ist fehlerhaft (z.B. ungültige Parameter, ungültige Syntax, ...). Als Lösung kann man prüfen, ob die URL richtig ist und die Parameter korrekt sind (eventuell sogar Browser-Cache leeren).
  4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
    • Keine Zugriffsberechtigung auf die angeforderte Ressource. Als Lösung könnte man wieder prüfen, ob die URL korrekt ist und ob die erforderlichen Berechtigungen vorhanden sind.
  5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performancegründen soll die Abfrage aber cacheable sein. Wie könnten Sie dafür eine Lösung angehen?
    • Man kann den "Cache-Control-Header" zur Website hinzufügen, welcher dann dafür sorgen kann, bestimmte Daten im Cache zu speichern und auszulesen.

1.6 HTML-Literatur lesen und Fragen beantworten

Was ist HTML?

HTML (Hyper Text Markup Language) ist die Standard-Sprache, um Websiten zu erstellen. Es beschreibt die Struktur einer Website und besteht aus vielen Elementen, die dem Internet Browser zeigen, wie der Inhalt der Seite dargestellt werden soll.

Wie kann man eine geschachtelte geordnete Liste der Tiefe 3 erzeugen?

  • <ol>: Startet eine geordnete Liste
  • <li>: Ein Element der Liste
Geordnete Liste der Tiefe 3:

<ol>
    <li>Ebene 1
      <ol>
        <li>Ebene 2
          <ol>
            <li>Ebene 3</li>
          </ol>
        </li>
      </ol>
    </li>
  </ol>

Wie ist eine HTML-Tabelle aufgebaut?

  • <table>: Zeigt, dass es eine Tabelle ist
  • <tr>: Erstellt eine Zeile
  • <th>: Erstellt eine Titel-Zelle
  • <td>: Erstellt eine Zelle
Beispieltabelle:

<table>
    <tr>
      <th>Spalte 1</th>
      <th>Spalte 2</th>
    </tr>
    <tr>
      <td>Zeile 1.1</td>
      <td>Zeile 1.2</td>
    </tr>
    <tr>
      <td>Zeile 2.1</td>
      <td>Zeile 2.2</td>
    </tr>
  </table>

Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?

Man sollte am besten relative Pfade verwenden. Wenn man lokal oder über die Domain auf die Website zugreift, sollten in beiden Fällen die Dateien korrekt geladen werden. Selbst bei einem Wechsel der Domain sollte es keine Probleme geben. Zusätzlich gilt:

  • Dateibenennung
    • klare Namen
    • keine Leerzeichen
    • in Kleinbuchstaben
    • keine Sonderzeichen
  • Ordnerstruktur
    • Alle generellen Seiten werden im root-Ordner erstellt
    • Unterseiten eventuell in eigenen Ordnern (z.B. Seiten zu den einzelnen Produkten im Ordner "Produkte")
    • eigener Ordner für Ressourcen (z.B. Bilder)
    • eigener Ordner für CSS-Stylesheets

Wie baut man in HTML ein Menü?

Man nutzt einfach eine ungeordnete Liste und passt sie mithilfe CSS-Attributen so an, sodass sie horizontal angezeigt wird und das Aufzählungszeichen entfernt wird. Dann kann man bei jedem Listenelement einen Link hinterlegen und schon hat man ein klassisches Webseitenmenü.

Beispiel:

CSS:

ul {
    list-style-type: none;
    margin: 0;
    padding: 0;
    overflow: hidden;
}

li {
    float: left;
}

li a {
    padding: 16px;
}

HTML:

<ul>
    <li><a href="/index.html">Startseite</a></li>
    <li><a href="/products.html">Produkte</a></li>
    <li><a href="/contact.html">Kontakt</a></li>
</ul>

Welche Attribute sollte man bei Bildern wie verwenden?

  • src: Pfad/URL zum Bild
  • alt: Alternative textuelle Beschreibung des Bildes
  • width: Breite des Bildes
  • height: Höhe des Bildes
